## P1 — 5xx responses leaking agent error text

### Where

`backend/routers/cockpit_agent.py` lines 97, 100, 138, 141, 177, 180.

### Why it is a violation

The agent returns a dict like `{"status": "error", "error": "<agent message>"}`.
These handlers pass `response.get("error")` directly into `HTTPException.detail`, which
means an uncontrolled string from the remote agent is forwarded to the HTTP client.
CLAUDE.md §Security: "Never put raw exception text in HTTPException(detail=…) for 5xx".

The existing `check_http_500_leaks.py` script does **not** catch this pattern because its
`_LEAK_PATTERNS` only match `str(e)`, `f"...{e}..."`, and `error_msg`. It silently misses
`response.get(...)`.

### Before

```python
# cockpit_agent.py — same pattern repeated at lines 97/100, 138/141, 177/180

if response.get("status") == "timeout":
    raise HTTPException(status_code=504, detail=response.get("error"))

if response.get("status") == "error":
    raise HTTPException(status_code=500, detail=response.get("error"))
```

### After

```python
from core.safe_http_errors import raise_internal_server_error

if response.get("status") == "timeout":
    logger.warning(
        "Agent %s timed out: %s", agent_id, response.get("error")
    )
    raise HTTPException(
        status_code=504,
        detail="Agent did not respond within the timeout",
    )

if response.get("status") == "error":
    logger.error(
        "Agent %s returned error: %s", agent_id, response.get("error")
    )
    raise HTTPException(
        status_code=500,
        detail="Agent returned an error",
    )
```

> Use `logger.warning` for 504 (expected timeout) and `logger.error` for 500 (unexpected
> failure). Do NOT use `raise_internal_server_error` here because it always raises 500 — the
> 504 case needs a manual `HTTPException`. The pattern above is correct.

### Files to change

- `backend/routers/cockpit_agent.py` — 6 occurrences (3 endpoint functions, each with a
  timeout + error check pair)

## P1 — Fix `check_http_500_leaks.py` blind spot

### Where

`backend/scripts/check_http_500_leaks.py` — the `_LEAK_PATTERNS` tuple.

### What is missing

The script does not detect `detail=response.get(...)` or `detail=response["key"]` patterns
inside 5xx `HTTPException` blocks.

### Before

```python
_LEAK_PATTERNS = (
    re.compile(r"detail\s*=\s*str\s*\(", re.I),
    re.compile(r"detail\s*=\s*error_msg\b"),
    re.compile(r"detail\s*=\s*f[\"'][^\"']*\{(?:e|exc|error)\}[^\"']*[\"']"),
    re.compile(
        r"detail\s*=\s*f[\"'][^\"']*\{str\s*\(\s*(?:e|exc|error)\s*\)\}[^\"']*[\"']"
    ),
)
```

### After

```python
_LEAK_PATTERNS = (
    re.compile(r"detail\s*=\s*str\s*\(", re.I),
    re.compile(r"detail\s*=\s*error_msg\b"),
    re.compile(r"detail\s*=\s*f[\"'][^\"']*\{(?:e|exc|error)\}[^\"']*[\"']"),
    re.compile(
        r"detail\s*=\s*f[\"'][^\"']*\{str\s*\(\s*(?:e|exc|error)\s*\)\}[^\"']*[\"']"
    ),
    # Catch dict lookups forwarding external/agent error strings into 5xx detail
    re.compile(r'detail\s*=\s*\w+\.get\s*\('),
    re.compile(r'detail\s*=\s*\w+\s*\[\s*["\']'),
)
```

> These new patterns are intentionally broad — any `response.get(...)` or `response["key"]`
> in a 5xx `HTTPException` is suspicious. If legitimate cases arise they can be added to the
> `ALLOW_LIST` with a justification comment.

### Files to change

- `backend/scripts/check_http_500_leaks.py`

### Where

`backend/services/inventory/query_service.py` — 839 lines, 18 methods, one class.

### Why it is a problem

The class mixes three distinct responsibilities:

1. **Cache management** — loading from Redis, parsing cache entries
   (`_parse_device_from_cache`, `_get_all_devices_cached`, `_get_custom_field_types`)

   (`_query_all_devices_live`, `_query_devices_by_location`, `_query_devices_by_ip_prefix`,
   `_query_devices_by_custom_field`)

   `_query_devices_by_tag`, `_query_devices_by_devicetype`, `_query_devices_by_manufacturer`,

**`device_cache.py`** — cache load and parse only:

```python
"""Redis device cache loader and parser for inventory queries."""
from __future__ import annotations
import logging
from typing import TYPE_CHECKING, Any, Dict, List, Optional

from models.inventory import DeviceInfo

if TYPE_CHECKING:
    from services.settings.cache import RedisCacheService

logger = logging.getLogger(__name__)
_BULK_CACHE_KEY = "nautobot:devices:all"


class DeviceCacheLoader:
    """Loads and parses the bulk device cache from Redis."""

    def __init__(self, cache_service: Optional[RedisCacheService] = None):
        self._cache_service = cache_service
        self._devices: Optional[List[DeviceInfo]] = None
        self._custom_field_types: Optional[Dict[str, str]] = None

    def invalidate(self) -> None:
        self._devices = None
        self._custom_field_types = None

    @staticmethod
    def parse_device(raw: Dict[str, Any]) -> DeviceInfo:
        tags = raw.get("tags") or []
        return DeviceInfo(
            id=raw.get("id", ""),
            name=raw.get("name"),
            serial=raw.get("serial"),
            primary_ip4=raw.get("primary_ip4"),
            status=raw.get("status"),
            device_type=raw.get("device_type"),
            role=raw.get("role"),
            location=raw.get("location"),
            platform=raw.get("platform"),
            tags=tags,
            manufacturer=raw.get("manufacturer"),
        )

    async def get_all(self) -> List[DeviceInfo]:
        """Return cached device list; populate from Redis on first call."""
        if self._devices is not None:
            return self._devices
        if self._cache_service is not None:
            try:
                raw_list = self._cache_service.get(_BULK_CACHE_KEY)
                if raw_list:
                    self._devices = [self.parse_device(d) for d in raw_list]
                    return self._devices
            except Exception:
                logger.warning("Redis cache unavailable, falling back to live query")
        return []  # caller falls back to live query when empty

    async def get_custom_field_types(self) -> Dict[str, str]:
        """Return custom field type map; cached per loader instance."""
        if self._custom_field_types is not None:
            return self._custom_field_types
        # ... existing fetch logic ...
        return self._custom_field_types
```

**`device_filters.py`** — pure stateless filter functions:

```python
"""Pure in-memory filter functions for DeviceInfo lists."""
from __future__ import annotations
from typing import List
from models.inventory import DeviceInfo


def by_name(devices: List[DeviceInfo], name_filter: str) -> List[DeviceInfo]:
    name_lower = name_filter.lower()
    return [d for d in devices if d.name and name_lower in d.name.lower()]


def by_role(devices: List[DeviceInfo], role_filter: str) -> List[DeviceInfo]:
    role_lower = role_filter.lower()
    return [d for d in devices if d.role and role_lower in d.role.lower()]


def by_status(devices: List[DeviceInfo], status_filter: str) -> List[DeviceInfo]:
    return [d for d in devices if d.status == status_filter]


def by_tag(devices: List[DeviceInfo], tag_filter: str) -> List[DeviceInfo]:
    tag_lower = tag_filter.lower()
    return [
        d for d in devices
        if any(tag_lower in str(t).lower() for t in (d.tags or []))
    ]


def by_device_type(devices: List[DeviceInfo], devicetype_filter: str) -> List[DeviceInfo]:
    dt_lower = devicetype_filter.lower()
    return [d for d in devices if d.device_type and dt_lower in d.device_type.lower()]


def by_manufacturer(devices: List[DeviceInfo], manufacturer_filter: str) -> List[DeviceInfo]:
    mfr_lower = manufacturer_filter.lower()
    return [d for d in devices if d.manufacturer and mfr_lower in d.manufacturer.lower()]


def by_platform(devices: List[DeviceInfo], platform_filter: str) -> List[DeviceInfo]:
    plat_lower = platform_filter.lower()
    return [d for d in devices if d.platform and plat_lower in d.platform.lower()]


def by_has_primary_ip(devices: List[DeviceInfo], require_primary: bool) -> List[DeviceInfo]:
    if require_primary:
        return [d for d in devices if d.primary_ip4]
    return [d for d in devices if not d.primary_ip4]
```

**`session_registry.py`**:

```python
"""Thread-safe session cancellation registry for Netmiko operations."""
import threading
from typing import Set


class SessionRegistry:
    def __init__(self) -> None:
        self._cancelled: Set[str] = set()
        self._active: Set[str] = set()
        self._lock = threading.Lock()

    def register(self, session_id: str) -> None:
        with self._lock:
            self._active.add(session_id)

    def unregister(self, session_id: str) -> None:
        with self._lock:
            self._active.discard(session_id)
            self._cancelled.discard(session_id)

    def cancel(self, session_id: str) -> None:
        with self._lock:
            self._cancelled.add(session_id)

    def is_cancelled(self, session_id: str) -> bool:
        with self._lock:
            return session_id in self._cancelled
```

## P3 — Replace `detail=str(e)` in 4xx responses

### Where

~26 locations across:
- `backend/routers/settings/credentials.py` (7 occurrences)
- `backend/routers/settings/rbac.py` (6 occurrences)
- `backend/routers/checkmk/tag_groups.py` (5 occurrences)
- `backend/routers/checkmk/host_groups.py` (2+ occurrences)
- `backend/routers/settings/compliance/rules.py` (3 occurrences)

### Why

4xx responses are explicitly allowed to carry informative detail per CLAUDE.md. However,
`str(e)` leaks whatever the exception message happens to contain — SQLAlchemy error text,
internal table names, or query fragments. The fix is to raise typed service exceptions
with controlled messages, and map them in the router.

### Before

```python
# routers/settings/credentials.py
try:
    credential = credential_service.get_credential(cred_id, db)
except Exception as e:
    raise HTTPException(status_code=400, detail=str(e))
```

### After

**Step 1** — define typed exceptions in the service or a shared exceptions module:

```python
# services/settings/exceptions.py  (new file, or add to existing)
class CredentialNotFoundError(Exception):
    def __init__(self, cred_id: int) -> None:
        super().__init__(f"Credential {cred_id} not found")
        self.cred_id = cred_id


class CredentialAccessDeniedError(Exception):
    pass
```

**Step 2** — raise typed exceptions from the service:

```python
# services/settings/credential_service.py
def get_credential(self, cred_id: int, db) -> Credential:
    result = self._repo.get_by_id(cred_id, db)
    if result is None:
        raise CredentialNotFoundError(cred_id)
    return result
```

**Step 3** — map in the router:

```python
# routers/settings/credentials.py
from services.settings.exceptions import CredentialNotFoundError

try:
    credential = credential_service.get_credential(cred_id, db)
except CredentialNotFoundError as e:
    raise HTTPException(status_code=404, detail=str(e))
except Exception as e:
    raise_internal_server_error(logger, "Failed to get credential", e)
```

> This pattern means the error text in `detail` is always produced by our own code, not
> raw exception messages from SQLAlchemy or other internals.

### Suggested approach

Do this incrementally per-module; not a big-bang refactor. Prioritise the modules that
are most likely to leak SQL error text (`credentials.py`, `rbac.py`).
